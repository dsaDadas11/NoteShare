🔔 通知功能设计 — 类似小红书

当别人点赞或评论你的笔记时，显示小红点提示

功能概览

通知系统需要覆盖以下几个核心交互：

A

仅点赞 + 评论通知

只覆盖两种场景：有人点赞你的笔记、有人评论你的笔记。最简单，符合课设范围。

B

点赞 + 评论 + 关注通知

三种通知类型。有人关注你时也触发通知。功能更完整，工作量适中。

C

点赞 + 评论 + 关注 + 评论点赞

四种通知类型。还包括别人点赞你的评论。最接近小红书完整体验，但工作量最大。

技术实现方式

通知的实时性如何保证？

P

轮询（Polling）

客户端每隔 N 秒请求一次通知接口。实现简单，适合课设，但有一定延迟（5-15秒）。

W

WebSocket 实时推送

服务端主动推送通知，几乎零延迟。实现复杂，需要维护长连接，课设中属于"加分项"。
