🗄️ 后端设计

新增数据库表：notifications

字段类型说明
idBIGINT主键
typeVARCHARLIKE / COMMENT
sender_idBIGINT触发者（谁点赞/评论）
receiver_idBIGINT接收者（笔记作者）
note_idBIGINT关联的笔记
is_readBOOLEAN是否已读
created_atDATETIME创建时间

新增 API

📌 GET /api/notifications — 获取通知列表（分页）
📌 GET /api/notifications/unread-count — 获取未读数量
📌 PUT /api/notifications/read-all — 全部标记已读
🔌 WS /ws/notification — WebSocket 实时推送

触发时机

• 用户点赞笔记 → 向笔记作者推送 LIKE 通知
• 用户评论笔记 → 向笔记作者推送 COMMENT 通知
• 自己操作不触发通知（不给自己发）

📱 Android 端设计

新增模块：feature/notification/

• NotificationApi.kt — Retrofit 接口（列表 + 未读数 + 已读）
• NotificationRepository.kt — 数据仓库
• NotificationScreen.kt — 通知列表页面（时间线样式）
• NotificationViewModel.kt — 状态管理
• WebSocketClient.kt — WebSocket 客户端，监听通知推送

红点逻辑

• MainViewModel 持有未读通知数状态
• WebSocket 收到推送 → 未读数 +1 → BottomNavBar 显示红点
• 进入通知页 → 调用已读接口 → 红点消失

导航变更

• "我的" 页面增加"通知"入口（顶部铃铛图标 + 红点）
• 新增路由 notification → NotificationScreen
